Multi-Model Orchestration Layer (Phase 3)
PhoennixAI does not run on one AI model. It runs on four, each doing what it does best. The user never knows which model runs underneath. They just see ARIA, or GRID, or PULSE. The four models are the engines. PhoennixAI is the vehicle.

CLAUDE: Reasoning and Strategy
All complex thinking, document creation, agent management, strategy, and writing. The primary intelligence layer. Powers ARIA, SCRIBE, and all core strategic decision making. This is the brain.

KIMI: Deep Research and Analysis
Long document analysis, competitor research, bulk data processing, cost-effective high-volume tasks. Powers SIGNAL agent and deep research workflows. Kimi K2.5 handles work at 10x lower cost than Claude for research-heavy tasks.

MANUS: Autonomous Execution
Multi-step autonomous task execution. Browsing, form filling, file management, complex workflows without human oversight. Powers GRID's execution arm and SCOUT's outreach tasks. Manus does, it does not think. Claude thinks, Manus acts.

PERPLEXITY: Real-Time Web Intelligence
Live web search with source citations. Real-time market data, news monitoring, competitor tracking. Powers SIGNAL and SCOUT with up-to-the-minute intelligence. The Perplexity Computer feature adds real-world execution on top of search, similar to how Manus operates but with web-native intelligence.

How it works: A router (a simple piece of logic) receives a task and decides which model is best for it. Each task goes to one model. The results come back to your system and are presented to the user as their PhoennixAI agent responding. All four models run simultaneously.

Build order: Claude API first (Unlock 2). Perplexity second for real-time web. Manus third for execution tasks. Kimi fourth when cost-effective bulk research is needed.

The DevHub vs Devin AI
Devin AI: Autonomous AI software engineer. Writes code, debugs, deploys end to end. £400-£500/month. Best for teams with complex ongoing engineering work. Impressive but general-purpose.
