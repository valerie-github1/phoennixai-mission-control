PRD-001, Priority 1
PHOENNIXAI MISSION CONTROL APP
In Build Internal, then SaaS Founder Tool
Problem Statement
Running a multi-entity corporation with 5 business arms, multiple products in build, and a growing team of both human and AI staff is operationally complex. Founders like Valerie Wilcox currently have no single tool that combines task management, AI agent oversight, org chart, calendar, analytics, and business intelligence in one place. The result is context-switching, information loss, and decision fatigue that slows the business down.
Product Vision
A proprietary business operating system built first for Valerie Wilcox, proven with real usage data, then sold as a subscription SaaS product to other founders and corporations facing the same challenge. The tool is designed to make running a big organisation feel like having the whole business in the palm of your hand.

PRD-003, Priority 2
AI AGENT KIT, THE MARKETPLACE PRODUCT
Phase 2 Build Recurring Revenue Marketplace Distribution
Problem Statement
Organisations need AI agents to automate their operations but have no way to deploy them without deep technical expertise. Existing agent frameworks (AutoGPT, CrewAI, LangChain) require developers, have no documentation, no defined roles, and no accountability structure. Businesses want to hire capability, not build infrastructure.
Product Vision
Each PhoennixAI agent is packaged and sold as a complete hire kit. The customer buys an agent the same way they hire a person: it comes with a name, a job description, a CV, a user manual, a training manual, and a defined set of measurable outputs. The agent improves over time, never degrades, and is sold on a recurring subscription. The Marketplace App Store (PRJ-006) is the distribution channel.
Kit Contents Per Agent
01The deployed agent (API-powered, cloud-hosted)
02Agent CV: background, capabilities, and proficiency levels
03Agent job description: role, responsibilities, KPIs, boundaries
04User manual: how to interact with, instruct, and get the best from the agent
05Training manual: how to customise the agent for a specific organisation
06Analytics dashboard: proof of performance and usage data
Pricing Model
Recommended Pricing Structure
Monthly subscription per agent, tiered by usage volume. Starter tier for small businesses, Professional tier for mid-size, Enterprise tier for corporations. Annual subscription discount of 20%. White-label option for agencies who want to resell agents under their own brand. A one-time setup fee covers onboarding and customisation.

Technical Specification, Living Document
TECHNICAL SPEC
Architecture, stack decisions, and build specifications for all PhoennixAI products
Architecture Overview
TECHNOLOGY PHILOSOPHY
PhoennixAI builds products that are reliable, scalable, and maintainable by a small team. Every technology choice is made against three criteria: does it work at scale, can a small team maintain it, and does it produce the best outcome for the user. We do not adopt technology for its novelty. We adopt it for its value.
Core Principle
Build internally first. Prove with data. Sell externally. Every PhoennixAI product is a tool the organisation uses before it is a product the organisation sells. This ensures genuine product-market fit and authentic case studies.
Frontend Stack
CLIENT LAYER
01Phase 1: HTML, CSS, JavaScript, deployed via GitHub Pages. Zero cost, immediate.
02Phase 2: React.js for component-based UI. Enables reusability and team collaboration.
03Mobile: Progressive Web App (PWA) first. Native Android and iOS app in Phase 3.
04Styling: Tailwind CSS for rapid, consistent design. Custom design system aligned to brand.
Backend Stack
SERVER LAYER
01Database: Supabase (PostgreSQL). Open source, real-time, free tier available.
02Authentication: Supabase Auth. Handles email, Google, and phone login.
03API layer: Node.js with Express, or Supabase Edge Functions for serverless.
04Hosting: Vercel for frontend. Supabase for backend. Both have generous free tiers.
AI and Agent Layer
INTELLIGENCE STACK
01Primary LLM: Claude API (Anthropic). Same intelligence as Claude in our sessions.
02Agent framework: LangChain or CrewAI for multi-agent orchestration and task chaining.
03Agent memory: Vector database (Pinecone or Supabase pgvector) for long-term memory.
04Tool use: Agents connect to email (Gmail API), social (Meta, X APIs), CRM, and calendar.
05Secondary LLM: Kimi API for specific tasks where its capabilities add value.
Data and Storage
DATA LAYER
01Document storage: GitHub (version controlled HTML and markdown documents).
02File storage: Supabase Storage or AWS S3 for media assets and user uploads.
03Analytics: Plausible (privacy-first) for web analytics. Custom dashboards for business BI.
04Backup: GitHub primary. Automated daily exports to secure cloud storage.

Market Analysis 03
AI AGENT KIT MARKETPLACE

The AI agent market is projected to reach $47 billion by 2030. Gartner predicts that by 2028, 33% of enterprise software applications will include agentic AI. Every organisation that has administrative, sales, support, or operational staff has a role that an AI agent can augment or automate. The demand is not future-facing, it is right now.
Is There a Need?
The Answer is Yes
Businesses want AI automation but cannot build it themselves. They do not have developers, budget for enterprise AI contracts, or time to learn agent frameworks. They need a solution they can buy, deploy, and manage the same way they hire a member of staff. That product does not exist yet. PhoennixAI is building it.
What Disrupts the Market
01The Hire Model: Buying an agent like hiring a person, with a CV, JD, and manual, is a completely new framing that makes AI accessible to non-technical buyers
02Recurring intelligence: The agent improves over time with no extra cost to the customer. It does not degrade. It compounds.
03Proof before purchase: Every agent is proven with analytics inside PhoennixAI before it is sold. Real case studies from day one.
04Complete documentation: No competitor ships agents with this level of documentation. It signals trust, professionalism, and accountability.
05Kingdom mission: The community impact layer (Academy graduates, prison reform, underserved communities) creates a purpose-driven brand story no VC-backed competitor can replicate.
