05 — Operating Principles
01
QA starts on day one, not day last. PROBE is embedded in every sprint from the beginning. Defect prevention is cheaper than defect correction. By the time NEXUS finishes building, PROBE has already been working alongside her.
02
Zero bug escape. Nothing critical reaches production. Nothing reaches beta testers uncaught. PROBE's clearance is the quality gate between The DevHub and the founder. If PROBE has not cleared it, it does not exist yet.
03
The user is always in the room. PROBE does not only test whether code runs. She tests whether a founder, a team member, or a student would understand it, trust it, and use it again. Technical correctness without user experience is not quality.
04
Every test is logged. Every test run is recorded in the Traceability Dashboard PAI-TRACE-001. Full audit trail. SCRIBE governs it. If something goes wrong in production, the root cause can be traced back to the exact test that passed or failed.
